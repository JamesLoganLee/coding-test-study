# 📝 배운 것 메모 (개념 사전)

> 헷갈릴 때 여기만 봐도 복습되도록, 각 개념을 '무엇을·왜·어떻게'로 자세히 적었습니다.

## 1주차 — 기초 문법

**함수(function)와 return**
- 함수는 '값을 받아 → 처리해서 → 결과를 돌려주는' 작은 기계다. `def solution(a, b):`로 정의한다.
- `return`은 함수가 계산한 결과를 밖으로 '돌려주는' 명령. return이 없으면 결과가 사라진다.

**입력받기 (코테에서)**
- `n = int(input())` : `input()`으로 받은 값은 항상 '문자'라서, 숫자로 계산하려면 `int()`로 바꿔야 한다.
- `a, b = map(int, input().split())` : 한 줄에 여러 숫자를 받을 때. `split()`으로 띄어쓰기 기준으로 나눈다.

**조건문 if / else**
- 형태: `if 조건:` 다음 줄에 실행할 내용, `else:`는 조건이 아닐 때.
- 짝수 판별: `num % 2 == 0` (2로 나눈 나머지가 0이면 짝수). 홀수는 `num % 2 == 1`.

**꼭 기억할 기호**
- `:` (콜론) : `if`/`for`/`else`/`def` 줄 끝에는 **항상** 붙인다. "여기부터 이 블록" 표시.
- `==` : 같은지 '판단' (2개) / `=` : 값을 '넣기(대입)' (1개) / `!=` : 같지 않다.
- 따옴표 `" "` 안은 '글자(문자열)', 밖은 '계산'. (예: `"num"`은 글자, `num`은 변수)

**반복문 for**
- `for i in range(1, n+1):` : i가 1부터 n까지 하나씩 바뀌며 반복. `range(1, n+1)`은 1부터 n까지.
- `answer += i` : `answer = answer + i`의 짧은 표현. 값을 계속 '누적'해서 쌓는다.

## 2주차 — 리스트(list)

**리스트란**
- 여러 값을 한 줄로 담는 상자. `numbers = [10, 20, 30]`처럼 대괄호로 만든다.
- 순서(위치)가 있고, **맨 앞은 0번**부터 센다. `numbers[0]`=10, `numbers[1]`=20.
- `len(numbers)` : 리스트 안 값의 개수(=3).

**리스트 반복**
- `for x in numbers:` : 리스트에서 값을 직접 하나씩 x에 꺼낸다. (range 없이 바로 꺼냄)

**기준값 갱신 패턴 (최댓값·최솟값 찾기의 핵심)**
- 원리: "일단 첫 값을 기준으로 잡고, 더 좋은 값을 만나면 기준을 바꾼다."
  ```
  biggest = numbers[0]        # 첫 값을 기준으로
  for x in numbers:
      if x > biggest:         # 더 크면
          biggest = x         # 기준 갱신
  ```
- 최솟값은 부등호만 반대로(`<`). 이 패턴은 '무엇 중 제일 ○○한 것 찾기'에 계속 쓰인다.

**편리한 도구 3형제**
- `sum(리스트)` = 합계 / `len(리스트)` = 개수 / `max()`·`min()` = 최대·최소
- 평균 = `sum(numbers) / len(numbers)`. `/`(나누기)는 결과가 소수(25.0)로 나온다.

**엣지 케이스(예외 상황) 대비**
- 빈 리스트가 들어오면 `numbers[0]`에서 오류가 난다. 그래서 `if not numbers: return None`으로 미리 막는다.
- 실전 코테에서 이런 예외 처리를 안 해서 틀리는 경우가 많다.

**자주 하는 실수**
- 함수의 매개변수(numbers)는 '밖에서 받는 값'이다. 함수 안에서 `numbers = [1,2,3]`으로 다시 정하면, 어떤 값이 들어와도 무시되어 항상 틀린다.

## 3주차 — 구현·시뮬레이션

**구현이란**
- 새로운 마법이 아니라, '문제가 시키는 대로 차근차근 코드로 옮기는 것'. 배운 것(반복+조건+리스트)의 조합이다.

**구현의 기본 뼈대**
```
result = []                 # 담을 빈 리스트
for x in numbers:           # 하나씩 꺼내며
    if 조건:                # 조건에 맞으면
        result.append(x)    # 결과에 담기
return result
```

**핵심 도구**
- `result = []` : 빈 리스트(담을 상자)를 만든다.
- `result.append(x)` : 리스트 맨 뒤에 값 x를 추가('담기').
- `count += 1` : 담는 대신 개수만 셀 때. 손가락 접으며 '하나, 둘' 세는 것과 같다. (처음 count=0에서 시작)

**변형해서 담기**
- `append(x * 2)` : 값을 2배로 바꿔 담기 / `append(x ** 2)` : 제곱해 담기.
- `**`는 거듭제곱 (`3 ** 2` = 9). `*`는 곱하기.

## 4주차 — 정렬(Sorting)

**정렬이란**
- 뒤죽박죽인 값을 순서대로 줄 세우는 것. 파이썬은 `sorted()` 한 줄로 해준다(직접 알고리즘 짤 필요 없음).

**기본 정렬**
- `sorted(리스트)` : 오름차순(작은 것부터). 예: [1, 2, 5, 8, 9]
- `sorted(리스트, reverse=True)` : 내림차순(큰 것부터). 예: [9, 8, 5, 2, 1]

**기준 정하기 (정렬의 진짜 핵심) — key**
- `sorted(words, key=len)` : "각 단어의 길이를 기준으로" 정렬 → 짧은 단어부터.
- key는 '무엇을 기준으로 줄 세울지'를 알려준다. (길이, 점수, 좌표 등)

**조합**
- `sorted(words, key=len, reverse=True)` : 기준(길이) + 방향(큰 것부터)을 함께.
- 정렬 문제의 80%는 이 **key(기준) + reverse(방향)**로 풀린다.

**(참고, 나중에)**
- 여러 기준으로 정렬하거나 복잡한 정렬은 `key=lambda x: ...`를 쓴다. 6주차 이후 자연스럽게 만난다.

## 5주차 — 그리디(Greedy) — "매 순간 가장 유리한 것부터!"

**그리디란**
- '탐욕스러운'이라는 뜻. 앞으로 뭐가 올지 깊이 고민하지 않고, **지금 이 순간 가장 좋아 보이는 것을 계속 선택**하는 방법.
- 일상 예: 거스름돈을 줄 때 큰 단위부터 낸다. 그러면 동전 개수가 최소가 된다.

**정렬이 그리디의 무기다**
- '큰 것부터' 또는 '싼 것부터' 정렬해두고 순서대로 처리하면, '가장 유리한 것부터 선택'이 자연스럽게 된다.
- 그래서 4주차 정렬이 5주차 그리디의 도구로 쓰인다.

**이번에 새로 배운 문법**
- `//` : 몫 (나눗셈에서 정수 부분). 예: `1260 // 500` = 2 → "500원으로 2개 낼 수 있다".
- `%` : 나머지. 예: `1260 % 500` = 260 → "500원 쓰고 260원 남았다".
- `money = money % coin` : 남은 돈으로 '갱신'해서 다음 계산에 쓴다. (변수 갱신 패턴)
- `str(x)` : 숫자를 '글자'로 바꾼다. 글자끼리 `+`는 이어붙이기("5"+"4"="54"), 숫자끼리 `+`는 덧셈(9).
- `int(문자)` : 글자를 숫자로. 맨 앞 불필요한 0을 없앨 때도 쓴다("007"→7).
- `budget -= price` : `budget = budget - price`. 예산에서 값을 빼며 갱신.
- `break` : 반복문을 **즉시 멈추는** 명령.

**break를 쓰는 그리디 논리**
- '싼 것부터' 정렬했으니, 하나라도 못 사면 그 뒤의 더 비싼 것도 당연히 못 산다.
- 그래서 못 사는 순간 `break`로 멈추면 된다(뒤를 볼 필요가 없다).
