# 📝 배운 것 메모 (개념 사전)

> 헷갈릴 때 여기만 봐도 복습되도록, 각 개념을 '무엇을·왜·어떻게'로 자세히 적었습니다.

## 1주차 — 기초 문법

**함수(function)와 return**
- 함수는 '값을 받아 → 처리해서 → 결과를 돌려주는' 작은 기계다. `def solution(a, b):`로 정의한다.
- `return`은 함수가 계산한 결과를 밖으로 '돌려주는' 명령. return이 없으면 결과가 사라진다.

**입력받기 (코테에서)**
- `n = int(input())` : `input()`으로 받은 값은 항상 '문자'라서, 숫자로 계산하려면 `int()`로 바꿔야 한다.
- `a, b = map(int, input().split())` : 한 줄에 여러 숫자를 받을 때. `split()`으로 띄어쓰기 기준으로 나눈다.

**조건문 if / else**
- 형태: `if 조건:` 다음 줄에 실행할 내용, `else:`는 조건이 아닐 때.
- 짝수 판별: `num % 2 == 0` (2로 나눈 나머지가 0이면 짝수). 홀수는 `num % 2 == 1`.

**꼭 기억할 기호**
- `:` (콜론) : `if`/`for`/`else`/`def` 줄 끝에는 **항상** 붙인다. "여기부터 이 블록" 표시.
- `==` : 같은지 '판단' (2개) / `=` : 값을 '넣기(대입)' (1개) / `!=` : 같지 않다.
- 따옴표 `" "` 안은 '글자(문자열)', 밖은 '계산'. (예: `"num"`은 글자, `num`은 변수)

**반복문 for**
- `for i in range(1, n+1):` : i가 1부터 n까지 하나씩 바뀌며 반복. `range(1, n+1)`은 1부터 n까지.
- `answer += i` : `answer = answer + i`의 짧은 표현. 값을 계속 '누적'해서 쌓는다.

## 2주차 — 리스트(list)

**리스트란**
- 여러 값을 한 줄로 담는 상자. `numbers = [10, 20, 30]`처럼 대괄호로 만든다.
- 순서(위치)가 있고, **맨 앞은 0번**부터 센다. `numbers[0]`=10, `numbers[1]`=20.
- `len(numbers)` : 리스트 안 값의 개수(=3).

**리스트 반복**
- `for x in numbers:` : 리스트에서 값을 직접 하나씩 x에 꺼낸다. (range 없이 바로 꺼냄)

**기준값 갱신 패턴 (최댓값·최솟값 찾기의 핵심)**
- 원리: "일단 첫 값을 기준으로 잡고, 더 좋은 값을 만나면 기준을 바꾼다."
  ```
  biggest = numbers[0]        # 첫 값을 기준으로
  for x in numbers:
      if x > biggest:         # 더 크면
          biggest = x         # 기준 갱신
  ```
- 최솟값은 부등호만 반대로(`<`). 이 패턴은 '무엇 중 제일 ○○한 것 찾기'에 계속 쓰인다.

**편리한 도구 3형제**
- `sum(리스트)` = 합계 / `len(리스트)` = 개수 / `max()`·`min()` = 최대·최소
- 평균 = `sum(numbers) / len(numbers)`. `/`(나누기)는 결과가 소수(25.0)로 나온다.

**엣지 케이스(예외 상황) 대비**
- 빈 리스트가 들어오면 `numbers[0]`에서 오류가 난다. 그래서 `if not numbers: return None`으로 미리 막는다.
- 실전 코테에서 이런 예외 처리를 안 해서 틀리는 경우가 많다.

**자주 하는 실수**
- 함수의 매개변수(numbers)는 '밖에서 받는 값'이다. 함수 안에서 `numbers = [1,2,3]`으로 다시 정하면, 어떤 값이 들어와도 무시되어 항상 틀린다.

## 3주차 — 구현·시뮬레이션

**구현이란**
- 새로운 마법이 아니라, '문제가 시키는 대로 차근차근 코드로 옮기는 것'. 배운 것(반복+조건+리스트)의 조합이다.

**구현의 기본 뼈대**
```
result = []                 # 담을 빈 리스트
for x in numbers:           # 하나씩 꺼내며
    if 조건:                # 조건에 맞으면
        result.append(x)    # 결과에 담기
return result
```

**핵심 도구**
- `result = []` : 빈 리스트(담을 상자)를 만든다.
- `result.append(x)` : 리스트 맨 뒤에 값 x를 추가('담기').
- `count += 1` : 담는 대신 개수만 셀 때. 손가락 접으며 '하나, 둘' 세는 것과 같다. (처음 count=0에서 시작)

**변형해서 담기**
- `append(x * 2)` : 값을 2배로 바꿔 담기 / `append(x ** 2)` : 제곱해 담기.
- `**`는 거듭제곱 (`3 ** 2` = 9). `*`는 곱하기.

## 4주차 — 정렬(Sorting)

**정렬이란**
- 뒤죽박죽인 값을 순서대로 줄 세우는 것. 파이썬은 `sorted()` 한 줄로 해준다(직접 알고리즘 짤 필요 없음).

**기본 정렬**
- `sorted(리스트)` : 오름차순(작은 것부터). 예: [1, 2, 5, 8, 9]
- `sorted(리스트, reverse=True)` : 내림차순(큰 것부터). 예: [9, 8, 5, 2, 1]

**기준 정하기 (정렬의 진짜 핵심) — key**
- `sorted(words, key=len)` : "각 단어의 길이를 기준으로" 정렬 → 짧은 단어부터.
- key는 '무엇을 기준으로 줄 세울지'를 알려준다. (길이, 점수, 좌표 등)

**조합**
- `sorted(words, key=len, reverse=True)` : 기준(길이) + 방향(큰 것부터)을 함께.
- 정렬 문제의 80%는 이 **key(기준) + reverse(방향)**로 풀린다.

**(참고, 나중에)**
- 여러 기준으로 정렬하거나 복잡한 정렬은 `key=lambda x: ...`를 쓴다. 6주차 이후 자연스럽게 만난다.

## 5주차 — 그리디(Greedy) — "매 순간 가장 유리한 것부터!"

**그리디란**
- '탐욕스러운'이라는 뜻. 앞으로 뭐가 올지 깊이 고민하지 않고, **지금 이 순간 가장 좋아 보이는 것을 계속 선택**하는 방법.
- 일상 예: 거스름돈을 줄 때 큰 단위부터 낸다. 그러면 동전 개수가 최소가 된다.

**정렬이 그리디의 무기다**
- '큰 것부터' 또는 '싼 것부터' 정렬해두고 순서대로 처리하면, '가장 유리한 것부터 선택'이 자연스럽게 된다.
- 그래서 4주차 정렬이 5주차 그리디의 도구로 쓰인다.

**이번에 새로 배운 문법**
- `//` : 몫 (나눗셈에서 정수 부분). 예: `1260 // 500` = 2 → "500원으로 2개 낼 수 있다".
- `%` : 나머지. 예: `1260 % 500` = 260 → "500원 쓰고 260원 남았다".
- `money = money % coin` : 남은 돈으로 '갱신'해서 다음 계산에 쓴다. (변수 갱신 패턴)
- `str(x)` : 숫자를 '글자'로 바꾼다. 글자끼리 `+`는 이어붙이기("5"+"4"="54"), 숫자끼리 `+`는 덧셈(9).
- `int(문자)` : 글자를 숫자로. 맨 앞 불필요한 0을 없앨 때도 쓴다("007"→7).
- `budget -= price` : `budget = budget - price`. 예산에서 값을 빼며 갱신.
- `break` : 반복문을 **즉시 멈추는** 명령.

**break를 쓰는 그리디 논리**
- '싼 것부터' 정렬했으니, 하나라도 못 사면 그 뒤의 더 비싼 것도 당연히 못 산다.
- 그래서 못 사는 순간 `break`로 멈추면 된다(뒤를 볼 필요가 없다).

## 6주차 — 이분탐색 & 투 포인터 — "범위를 좁혀가며 찾기"

### 🔍 이분탐색 (Binary Search)

**이분탐색이란**
- 두꺼운 사전에서 단어를 찾을 때, 앞에서부터 한 장씩 넘기지 않는다. **가운데를 펴서 → 앞쪽이면 앞 절반만, 뒤쪽이면 뒤 절반만** 다시 본다. 이걸 반복하는 것이 이분탐색이다.
- 매번 **절반을 버리므로 매우 빠르다**. 100만 개 데이터도 약 20번이면 찾는다. (하나씩 확인하면 최대 100만 번)

**★ 전제조건: 반드시 정렬되어 있어야 한다**
- 정렬돼 있어야 "가운데 값보다 크면 오른쪽에 있다"는 판단이 가능하다.
- 정렬 안 된 리스트에서는 이 방법을 쓸 수 없다. (그래서 4주차 정렬이 계속 도구로 쓰인다)

**핵심 3줄**
```
mid = (left + right) // 2   # 가운데 위치 계산
left = mid + 1              # 왼쪽 절반 버리기 (찾는 값이 오른쪽에 있을 때)
right = mid - 1             # 오른쪽 절반 버리기 (찾는 값이 왼쪽에 있을 때)
```

**실제 흐름 예시** — numbers=[1,3,5,7,9], target=7
| 회차 | 범위(left~right) | mid | numbers[mid] | 판단 | 조치 |
|------|-----------------|-----|--------------|------|------|
| 1회 | 0~4 | 2 | 5 | 5 < 7 → 오른쪽에 있다 | left=3 (1,3,5 버림) |
| 2회 | 3~4 | 3 | **7** | 찾았다! | return 3 ✅ |
- 단 2번 만에 찾았다. 1회차에서 **왼쪽 절반(3개)을 통째로 버린 것**이 핵심.

**`left = mid + 1`의 의미**
- "가운데(mid)는 이미 확인했고 아니었으니, **그 다음(mid+1)부터** 오른쪽만 보자"는 뜻. 왼쪽 절반과 mid까지 한 번에 버린다.

### 👆 투 포인터 (Two Pointers)

**투 포인터란**
- 양쪽 끝에서 두 사람(손가락)이 서로를 향해 걸어오는 방식. **왼쪽(left)과 오른쪽(right)에 손가락을 두고, 상황에 따라 한쪽씩 안으로 옮기며** 답을 찾는다.

**이분탐색과의 차이**
| | 이분탐색 | 투 포인터 |
|---|---------|-----------|
| 방식 | 가운데를 보고 **절반을 버림** | 양 끝 두 개를 보고 **한쪽을 옮김** |
| 용도 | 하나의 **값 찾기** | 두 값의 **조합 찾기** / 앞뒤 **비교** |
| 공통 | 둘 다 **정렬된** 리스트에서 사용 | |

**움직이는 논리 (정렬되어 있으니 가능)**
- **합이 작다** → 더 큰 값이 필요 → `left += 1` (오른쪽으로, 오른쪽일수록 큰 값)
- **합이 크다** → 더 작은 값이 필요 → `right -= 1` (왼쪽으로, 왼쪽일수록 작은 값)

**실제 흐름 예시** — numbers=[1,3,5,7,9], target=12
| 회차 | left(값) | right(값) | 합 | 판단 | 조치 |
|------|---------|-----------|-----|------|------|
| 1회 | 0 (1) | 4 (9) | 10 | 10 < 12 (작다) | left=1 |
| 2회 | 1 (3) | 4 (9) | **12** | 딱 맞다! | return True ✅ |

**회문(팰린드롬) 판별에도 활용**
- 양 끝 글자를 비교하며 안쪽으로 좁혀온다. **다르면 즉시 False**, 끝까지 같으면 True.
- `word[left]` — 문자열도 리스트처럼 **위치로 글자를 꺼낼 수 있다**. `"level"[0]` = `'l'`

### 🆕 6주차 새 문법

**`while` 반복문**
- `while 조건:` — **조건이 참인 동안 계속** 반복한다.
- `for`와의 차이: `for`는 정해진 횟수(리스트 개수)만큼, `while`은 **횟수를 모를 때**(조건이 깨질 때까지).
- 이분탐색은 몇 번 반복할지 모르므로 `while`을 쓴다.

**`elif`**
- "else if"의 줄임말. 조건이 **3개 이상**일 때 `if` → `elif` → `else` 순서로 차례로 확인한다.
- 위에서부터 보다가 맞는 조건을 만나면 거기서 끝 (나머지는 확인 안 함).

**★ 들여쓰기 = 문법 (이번 주 최대 수확)**
- 파이썬은 **줄 앞의 띄어쓰기로 "이 명령이 누구에게 속하는지"**를 결정한다. (다른 언어의 `{ }` 역할)
```
while 조건:              ← 1단계
    이건 while 안        ← 2단계 (while에 속함)
    if 조건:             ← 2단계
        이건 if 안       ← 3단계
이건 while 밖            ← 1단계 (while이 다 끝난 뒤 실행)
```
- **자주 하는 실수**: `if`를 `while` 밖에 두면 → `while`이 아무것도 안 바꾸고 **무한 반복**(무한루프)에 빠진다.
- 회문 판별에서 `left += 1`, `right -= 1`은 **`if` 밖, `while` 안**에 둬야 한다 (조건과 상관없이 매번 좁혀야 하므로).

## 7주차 — 스택 · 큐 · 해시 — "새로운 3가지 저장 도구"

### 📚 스택 (Stack)

**스택이란**
- 접시 쌓기와 같다. 쌓으면(넣으면) 맨 위에 쌓이고, 뺄 때도 **맨 위(마지막에 넣은 것)부터** 뺀다. 아래 접시를 먼저 못 뺀다.
- **LIFO (Last In, First Out)** = 마지막에 넣은 게 먼저 나온다.

**기본 사용법**
```python
stack = []
stack.append(1)    # 넣기(쌓기) — 3주차에 배운 append와 동일
stack.pop()          # 맨 뒤(마지막) 값을 꺼내면서 제거
```

**왜 괄호 검사에 스택을 쓰나**
- 괄호는 '가장 최근에 연 것부터 닫혀야' 짝이 맞는다(안쪽부터 닫힘). 그래서 여는 괄호를 스택에 쌓아두고, 닫을 때마다 **가장 최근 것**과 짝을 맞춘다.

**실제 흐름 예시** — "(())"
| 글자 | 판단 | 스택 상태 |
|------|------|-----------|
| `(` | 여는 괄호 → 쌓기 | `[(]` |
| `(` | 여는 괄호 → 쌓기 | `[(, (]` |
| `)` | 닫는 괄호 → 짝 찾아 꺼내기 | `[(]` |
| `)` | 닫는 괄호 → 짝 찾아 꺼내기 | `[]` |
- 끝나고 스택이 비었으니 → True. 만약 글자가 다 끝났는데 스택에 뭔가 남아있으면(예: "(()") → 짝 안 맞은 여는 괄호가 있다는 뜻 → False.

---

### 🚶 큐 (Queue)

**큐란**
- 은행 번호표 줄과 같다. **먼저 온 사람이 먼저** 처리된다. 스택과 정반대!
- **FIFO (First In, First Out)** = 먼저 넣은 게 먼저 나온다.

**기본 사용법**
```python
from collections import deque   # 큐를 위한 도구를 불러옴
queue = deque()
queue.append(x)      # 뒤에 넣기 (스택과 동일)
queue.popleft()       # 맨 앞(가장 먼저 넣은 것)을 꺼내면서 제거 (스택의 pop()과 다름!)
```

**스택 vs 큐 — 핵심 비교**
| | 스택 | 큐 |
|---|------|-----|
| 비유 | 접시 쌓기 | 번호표 줄서기 |
| 꺼내는 순서 | 마지막에 넣은 것부터 (LIFO) | 처음에 넣은 것부터 (FIFO) |
| 꺼내기 명령 | `pop()` | `popleft()` |

**`while queue:` 의 의미**
- `queue` 안에 값이 남아있으면 참(True), 텅 비면(`[]`) 거짓(False)으로 취급된다.
- "줄에 사람이 남아있는 동안 계속 부르고, 다 부르면 멈춰라"는 뜻.

---

### 🔑 해시 / 딕셔너리 (dict)

**딕셔너리란**
- 국어사전과 같다. '단어(키, key)'를 찾으면 그 '뜻(값, value)'이 바로 나온다. 처음부터 다 뒤지지 않고 **키만 알면 바로** 값을 찾는다.
- 리스트는 `numbers[0]`처럼 **번호(위치)**로 찾지만, 딕셔너리는 `scores["철수"]`처럼 **이름(키)**으로 찾는다. 그리고 훨씬 빠르다.

**기본 사용법**
```python
count = {}                # 빈 딕셔너리 만들기
count["사과"] = 3          # "사과"라는 키에 3을 저장
count["사과"]              # 3 (키로 값 바로 꺼내기)
"사과" in count             # 키가 있는지 확인 (있으면 True)
count.get("포도", 0)        # 키가 없으면 기본값 0을 줌 (오류 안 남)
```

**개수 세기의 정석 패턴 — "있으면 +1, 없으면 1로 시작"**
```python
if ch in count:        # 이미 나온 적 있는 키면
    count[ch] += 1       # 개수 +1
else:                    # 처음 나온 키면
    count[ch] = 1         # 개수를 1로 시작
```
- 이 패턴은 '개수 세기' 문제에서 정말 자주 쓰인다. 꼭 기억해둘 것.

**실제 흐름 예시** — "hello"
| 글자 | 판단 | count 상태 |
|------|------|------------|
| h | 처음 나옴 | `{'h': 1}` |
| e | 처음 나옴 | `{'h': 1, 'e': 1}` |
| l | 처음 나옴 | `{'h': 1, 'e': 1, 'l': 1}` |
| l | 또 나옴! `count['l'] += 1` | `{'h': 1, 'e': 1, 'l': 2}` |
| o | 처음 나옴 | `{'h': 1, 'e': 1, 'l': 2, 'o': 1}` |

**★ 들여쓰기 실수 사례 (직접 겪고 고친 것)**
- `return count`를 `for` 블록 **안**(if/else와 같은 들여쓰기)에 두면, 글자 하나를 처리하자마자 함수가 끝나버려서 **첫 글자 결과만** 나온다. (예: "hello" → `{'h': 1}`만 나오고 끝)
- 원인: `return`이 `else`와 같은 높이에 있으면, 파이썬은 "이건 여전히 for 반복문 안"이라고 이해해서 매 반복마다 함수를 끝내버린다.
- 해결: `return count`는 **for와 같은 들여쓰기**(= for 블록이 다 끝난 뒤, for 밖)에 둬야 한다.
- **기억할 규칙**: "반복문 안에서 다 처리하고 결과를 돌려줄 땐, `return`을 반복문과 같은 들여쓰기(반복문 밖)에 둔다."
